# LLM Fit Software Research

Исследование рынка инструментов, которые помогают ответить на вопрос:

> Какую LLM имеет смысл запускать на конкретном железе локально?

Фокус исследования:

- подбор моделей под RAM / VRAM / CPU / GPU
- оценка fit / speed / quality / context
- рекомендации по quantization и размеру модели
- смежные продукты, которые продолжают пользовательский путь после этапа выбора

## Краткий вывод

Рынок делится на 3 основных слоя:

1. `Fit estimators`
   Инструменты, которые пытаются ответить, какие модели вообще поместятся и будут работать на конкретной машине.

2. `Benchmark / evidence layer`
   Инструменты и базы данных, которые показывают не только теоретический fit, но и фактическую производительность на похожем железе.

3. `Execution platforms`
   Продукты, в которых пользователь уже скачивает, запускает и использует выбранную модель локально.

Самые близкие прямые аналоги для темы `llmfit`:

1. `llmfit`
2. `CanIRun.ai`
3. `LocalLLM.run`
4. `LocalScore`
5. `Jan`

## Практическая цель исследования

Это исследование делается не как академический обзор рынка, а как прикладная рабочая база для ответов на 3 вопроса:

1. Где на рынке уже есть сильные решения, которые не стоит дублировать.
2. В какой точке ещё остаётся незакрытая практическая ценность.
3. Как выглядел бы минимальный исследовательский MVP, который можно использовать для проверки спроса и product direction без написания полноценной платформы.

## Метод отбора

Кандидаты ранжировались по 4 критериям:

1. Близость к core-job: "что я могу запустить на этом железе?"
2. Зрелость продукта и признаки поддержки
3. Практическая полезность для локального AI workflow
4. Качество публичных материалов: README, docs, сайт
